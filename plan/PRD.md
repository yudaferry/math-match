# Product Requirements Document — Math Match (MVP)

## App Name

**Math Match**
Tagline: Latihan aritmatika cepat dan fokus.

---

## Overview

Math Match is a minimal, tool-like web app for children ages 4–12 to practice basic arithmetic. A player or mentor configures a session, the player answers questions one by one, and a result summary is shown at the end. No accounts, no scores, no levels — just focused arithmetic practice.

Language: Indonesian throughout.
Platform: Web (PWA), works offline for single-session use.

---

## User Flow

```
[Configuration Page]
        |
        | → Press "Mulai"

[Question Screen]  ←──────────────┐
        |                         |
        | → Submit answer         | (repeat until question count reached)
```

## Configuration Page

All fields must be filled before the start button is enabled.

### Fields

| Field | Type | Options / Range | Default |
|---|---|---|---|
| Operasi | Multi-select checkbox | Penjumlahan, Pengurangan, Perkalian, Pembagian | Penjumlahan |
| Jumlah Soal | Number input | 1 – 100 | 10 |
| Angka Minimum | Number input | 0 – 999 | 1 |
| Angka Maksimum | Number input | 1 – 999 | 10 |
| Hasil Negatif Diperbolehkan | Toggle | Ya / Tidak | Tidak |
| Sisa Pembagian Diperbolehkan | Toggle | Ya / Tidak | Tidak |
| Mode Timer | Select | Tanpa Timer, Stopwatch, Hitung Mundur per Soal, Hitung Mundur Total Sesi | Tanpa Timer |
| Durasi Timer | Number input (seconds) | 1 – 3600 | Visible only when a countdown mode is selected |

### Validation Rules

- Angka Maksimum must be greater than Angka Minimum.
- At least one Operasi must be selected.
- If "Pembagian" is selected and "Sisa Pembagian Diperbolehkan" is Tidak, the app must only generate divisible pairs at question generation time.
- If "Pengurangan" is selected and "Hasil Negatif Diperbolehkan" is Tidak, the app must ensure the second operand is always less than or equal to the first.
- Durasi Timer is required and must be > 0 if a countdown mode is selected.

### Start Button

Label: **Mulai**
Disabled until all validation passes.

## Question Screen

### Layout

The layout is responsive based on screen size:

**Smartphone (portrait, small screen)**
```
┌─────────────────────────┐
│  Soal 3 dari 10   ⏱ 12s │  ← top bar
├─────────────────────────┤
│                         │
│       7 × 8 = ?         │  ← question (large)
│       [  56  ]          │  ← answer display
│                         │
├─────────────────────────┤
│  [ 7 ] [ 8 ] [ 9 ]      │
│  [ 4 ] [ 5 ] [ 6 ]      │  ← numpad (bottom)
│  [ 1 ] [ 2 ] [ 3 ]      │
│  [ ← ] [ 0 ] [Jawab]    │
└─────────────────────────┘
```

**Tablet / Desktop (landscape, large screen)**
```
┌──────────────────────────────────────┐
│  Soal 3 dari 10              ⏱ 12s   │  ← top bar
├───────────────────┬──────────────────┤
│                   │  [ 7 ] [ 8 ] [ 9]│
│    7 × 8 = ?      │  [ 4 ] [ 5 ] [ 6]│
│    [  56  ]       │  [ 1 ] [ 2 ] [ 3]│
│                   │  [ ← ] [ 0 ][Jwb]│
└───────────────────┴──────────────────┘
```

**Numpad buttons:**
- Digits 0–9
- Backspace (←) — delete last digit
- Jawab — submit answer (also triggered by Enter key)

**Input rules:**
- No native keyboard shown on mobile (use `inputmode="none"` on the hidden input)
- Answer is built digit-by-digit via numpad taps
- Supports negative sign (-) only if subtraction-with-negative is enabled in config
- Max input length: 4 digits (sufficient for numbers up to 999 × 999 = no, max result is 999+999=1998, so 4 digits)

### Timer Behavior

| Mode | Behavior |
|---|---|
| Tanpa Timer | No timer shown |
| Stopwatch | Running clock counting up, visible. No penalty. |
| Hitung Mundur per Soal | Countdown per question. On expiry, question is marked wrong and next question loads automatically. |
| Hitung Mundur Total Sesi | Single countdown for the whole session. On expiry, session ends immediately and result screen shows with remaining questions marked wrong. |

### Question Generation

- Questions are randomly generated at session start based on configuration.
- All questions for the session are pre-generated (not generated on the fly) so the total count is always exact.
- Division-without-remainder: generated by picking a result first, then multiplying to get the dividend.
- Subtraction-without-negative: first operand is always >= second operand.

### Answer Handling

- Only numeric input accepted (integers).
- After submitting, no feedback is shown (correct/wrong not revealed per question — only in result).
- Auto-advance to next question after submit.

## Result Screen

### Displayed Information

- Jumlah soal dijawab benar (e.g., "8 dari 10 benar")
- Jumlah soal salah (e.g., "2 salah")
- Waktu total sesi (elapsed time in mm:ss format, always shown regardless of timer mode)
- Question review table: each question, the player's answer, the correct answer, and a correct/wrong indicator

### Actions

- **Ulangi** — restarts the session with the exact same configuration
- **Konfigurasi Baru** — returns to the configuration page

## Tech Stack

| Layer | Choice |
|---|---|
| Framework | Next.js 14 (App Router) |
| Styling | Tailwind CSS |
| Language | TypeScript |
| PWA | next-pwa or custom service worker via next.config |
| Offline storage | None for MVP (session state in memory only) |
| Deployment | Vercel (primary) or Cloudflare Pages |
| Package manager | npm or bun |

No backend, no database, no authentication for MVP.

---

## Out of Scope for MVP

- User accounts and login
- Progress tracking and history
- Multiple player profiles
- Rewards, achievements, or badges
- Animations and sound effects
- Mascots or illustrated characters
- Leaderboards or multiplayer/classroom mode
- Hints or step-by-step help
- Multi-language support (English or others)
- Embeddable widget
